Some notes I've written for myself doing os dev with RISC-V based on riscv-gnu-toolchain.

Tutorial

This is my learning process of following the wiki.osdev.org/Bare_Bones tutorial for RISC-V in RISC-V assembly.

In many ways writing the whole thing in assembly is much simpler, in the original tutorial it requires some assembly to bootstrap the C kernel file, while here we only need 1 file written in assembly. It also simplifies the build process.

This first file will be a minimal bare-bones kernel which writes to VGA and uses the multiboot standard.

.equ ALIGN, 1 << 0 /* align loaded modules on page boundaries */ .equ MEMINFO, 1 << 1 /* provide memory map */ .equ FLAGS, ALIGN | MEMINFO /* this is the Multiboot 'flag' field */ .equ MAGIC, 0x1BADB002 /* 'magic number' lets bootloader find the header */

RISC-V Assembler Directives

Assembler directives are like instructions but only exist at compile-time and are not translated into machine code. They are "directives" to tell the assembler what to do, like defining compile-time constants (e.g. find and replace every instance of this symbol with this value). You won't find assembler directives in the RISC-V ISA. These will basically always be prefixed with `.` e.g. `.equ` is a assembler directive while `add` is not. Assembler directives are not specific to RISC-V and are often shared across different ISAs.

Directive Description
.equ Defines a symbolic name for a value.
.section Defines a section that will be used when laying out the program in memory.
.align Aligns the next data to a given alignment boundary (in bytes). The given value must be a power of 2. It basically just adds padding until the next address is a multiple of the given value (`address % value == 0`).
.multiboot Used to define the multiboot header section for a multiboot-compliant bootloader. Usualy seen at the top of the program with `.section .multiboot`.
.long Writes a 32-bit value into memory. `.equ` defines a value but doesn't actually include it in the binary, `.long` does, so you will often see these used together.
.text Used to define the section where assembly instructions are placed for execution e.g. `.section .text`.
.bss Used to define the section where uninitialized data is placed in memory e.g. `.section .bss`.
.data Used to define the section where initialized data is placed in memory e.g. `.section .data`.
.global Used to define a symbol that is visible to the linker; so it exists after the assembler. E.g. `.global _start` where `_start` is the entry point of the program that needs to be visible to the linker.
.type Specifies the type of a symbol to help the linker distinguish between functions and data. E.g. `.type _start, @function`.
.size Specifies the size of a symbol in bytes. E.g. `.size _start, . - _start` sets the size of `_start` to the current location `.` minus `_start`; this effectively communicates that "this symbol occupies exactly the bytes from its label up to here" which is useful for debugging.

RISC-V Instructions

The instructions found in the RISC-V ISA.

Instruction x86 equivalent aarch64 equivalent Description Extension
add add add Add two registers. Base Integer
la mov Load address of a symbol into a register.
call call Calls a function; by pushing the return address onto the stack and jumping to the function's address.
csrrc Reads a CSR and clears bits in the register (csr-read).
wfi hlt Waits for interrupt.
j jmp Jumps to a label e.g. `j 1b`. Labels can be defined a few different ways, with `1b` saying to jump to the nearest `1` label before (back) this point, with `1f` saying to jump to the nearest `1` label after (forward) this point.
